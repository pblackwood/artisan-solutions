Solution Design & Architecture

My whole career has been about making sound engineering decisions. It's not about the newest or coolest technologies. It's about finding the solution you and your team can actually deliver in the time and budget you have.

It's always a balancing act between feature set and simplicity, build vs buy, performance vs rich user experience. I am good at managing the interplay of different priorities.

Clean, Well-Tested Code

Your applications won't do much without code. To insure the very highest quality, embrace testing in all environments, from the developer's laptop to production. Automated unit tests are a given. You also need automated UI testing, end-to-end system integration testing, security vulnerability testing, and of course good old manual testing by subject matter experts.

I can show you how to embed quality metrics into your team's process. Think continuous integration, code coverage, static code analysis, security vulnerability analysis and many more tools can be embedded into the SDLC with low impact and maximum benefit.
